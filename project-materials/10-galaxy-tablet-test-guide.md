# 갤럭시 태블릿 테스트 가이드

## 1. 가장 먼저 이해할 점

갤럭시 태블릿에서 이 앱을 `제대로` 테스트하려면
보통 `https://` 주소가 필요합니다.

이유:

- 카메라 권한은 보안 연결에서 더 안정적으로 동작
- PWA 설치도 HTTPS에서 더 잘 동작
- 서비스 워커도 보안 환경이 사실상 기준

쉽게 말하면:

`이 앱을 태블릿에서 실제처럼 써보려면 임시라도 HTTPS 주소를 하나 만드는 것이 가장 좋다`

## 2. 가장 쉬운 테스트 방식

### 준비물

- 현재 프로젝트 폴더
- Mac 한 대
- 갤럭시 태블릿
- 인터넷 연결

### 순서

1. Mac에서 `Build-Study-Helper-for-SA-Tablet-Release.command` 를 더블클릭합니다.
2. 그러면 `release/study-helper-for-sa-tablet-web/` 폴더가 만들어집니다.
3. 이 폴더를 정적 호스팅 서비스에 올립니다.
4. 생성된 `https://` 주소를 갤럭시 태블릿 Chrome에서 엽니다.
5. 카메라와 마이크 권한을 허용합니다.
6. 실제 문제집으로 기능을 하나씩 확인합니다.

## 2-1. 지금 바로 쓸 수 있는 임시 방식

정식 배포 전에
`임시 HTTPS 터널 주소`
를 열어서 먼저 확인할 수도 있습니다.

이 방식은:

- 빠르게 실기 확인 가능
- 주소가 임시일 수 있음
- Mac이 계속 켜져 있어야 함

즉, 이 방식은
`집에서 계속 쓰는 링크`
가 아니라
`빠르게 테스트하는 링크`
에 가깝습니다.

Mac 전원을 끄면 링크도 멈춥니다.

## 3. 왜 로컬 주소보다 HTTPS가 좋은가

Mac에서 `http://127.0.0.1:8010` 으로 보는 것은
Mac 자체 미리보기용입니다.

하지만 갤럭시 태블릿에서는:

- 같은 주소를 그대로 열 수 없음
- 로컬 네트워크 주소는 카메라/PWA에서 제한될 수 있음

그래서 태블릿 실사용 검증은
`정적 호스팅 + HTTPS`
방식이 가장 안전합니다.

## 4. 실제 테스트 순서

### 1단계. 첫 실행 확인

- 화면이 정상적으로 열리는지
- 상단 타이틀이 `Study Helper for SA` 하나로만 보이는지
- `For Tablet` 이 작은 보조 문구로만 보이는지
- 스크롤이 너무 불편하지 않은지

### 2단계. 카메라 확인

- `사진 찍기` 버튼을 누르면 전체화면 촬영이 열리는지
- 전체화면에서 문제집 화면이 넓게 보이는지
- `촬영하기` 를 누르면 메인 화면에 결과 미리보기가 남는지
- 한 문제만 크게 보일 때 더 잘 읽는지

### 3단계. 사진 업로드 확인

- 앨범에서 사진을 골라 불러올 수 있는지
- 찍은 직후 미리보기가 바뀌는지

### 4단계. OCR 확인

- `OCR 전체 읽기` 에 글자가 보이는지
- 문제 번호 칩이 생기는지
- 1차 읽기가 약할 때 다시 읽는 느낌으로 개선됐는지

### 5단계. 코칭 확인

- `3번 문제 도와줘` 입력 후 결과가 나오는지
- 힌트 1, 2, 3이 순서대로 열리는지
- 정답은 힌트를 다 열기 전에는 안 보이는지

### 6단계. 음성 확인

- `음성 입력` 버튼이 동작하는지
- 듣는 중 / 정리 중 상태가 버튼에서 보이는지
- 말한 문제 번호가 `문제 번호` 입력칸에도 들어가는지
- 힌트 카드 안 `자동 읽기` 가 동작하는지
- 마지막 단계 카드 안 `자동 읽기` 가 동작하는지

### 7단계. ChatGPT 보조 확인

- `ChatGPT 복사` 가 되는지
- `공유하기` 로 ChatGPT 앱이나 다른 앱으로 보낼 수 있는지

### 8단계. 기록 확인

- 문제를 한 번 분석하면 `최근 코칭 기록` 에 쌓이는지
- 기록을 누르면 다시 열리는지
- `기록 지우기` 가 되는지

## 5. 테스트할 때 준비하면 좋은 문제 예시

- 짧은 색깔 문제 1장
- 짧은 빈칸 문제 1장
- 번호가 2개 이상 있는 문제 페이지 1장

이 3가지를 보면
지금 앱이 어디까지 잘 되는지 빨리 파악할 수 있습니다.

## 6. 사진 찍는 팁

- 글자가 화면에서 작지 않게 찍기
- 그림보다 글자가 중심이 되게 찍기
- 빛 반사가 적은 곳에서 찍기
- 태블릿과 문제집을 최대한 평행하게 두기
- 한 번에 너무 많은 문제를 넣지 않기

## 6-1. PC 화면을 다시 찍는 테스트는 왜 약한가

휴대폰이나 태블릿으로
`PC 모니터 화면을 다시 찍는 방식`
은 OCR 정확도가 떨어지기 쉽습니다.

이유:

- 모니터 픽셀 무늬 때문에 글자가 깨져 보일 수 있음
- 화면 반사와 밝기 때문에 글자 경계가 흐려질 수 있음
- 카메라 각도가 조금만 틀어져도 글자가 찌그러짐
- 실제 문제집 종이보다 화면 글자가 더 작게 잡히는 경우가 많음

그래서 이 앱은
`문제집 종이 사진`
또는
`선명한 이미지 업로드`
기준으로 보는 것이 더 정확합니다.

가능하면:

- PC 화면을 찍기보다 원본 이미지를 바로 업로드
- 또는 문제집 종이를 직접 찍기

방식이 더 좋습니다.

## 7. 테스트 결과를 기록하는 방법

문제가 생기면 아래 4가지만 적어두면 됩니다.

- 어떤 문제집 페이지였는지
- 카메라로 찍었는지, 사진 업로드였는지
- 어떤 버튼을 눌렀는지
- 결과가 어떻게 이상했는지

예:

`영어 문제집 12쪽, 사진 업로드, 3번 문제 도와줘 입력, OCR이 8번으로 잘못 읽음`

## 8. 최실장님께 추천하는 실제 테스트 흐름

가장 쉬운 순서는 이렇습니다.

1. Mac에서 릴리스 폴더 생성
2. HTTPS 주소 하나 만들기
3. 갤럭시 태블릿 Chrome에서 열기
4. 카메라 권한 허용
5. 문제 3장 정도로 빠르게 점검
6. 잘 안 되는 케이스만 다시 메모
7. 그 메모를 기준으로 다음 튜닝 진행

## 9. 한 줄 정리

`태블릿 실사용 테스트는 HTTPS 주소를 하나 만든 뒤, 갤럭시 태블릿 Chrome에서 카메라, OCR, 힌트, 음성, 기록 기능을 순서대로 확인하는 방식으로 진행하면 된다`
