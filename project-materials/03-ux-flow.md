# UX 흐름

## 1. 태블릿 기준 핵심 흐름

### 흐름 A. 사진으로 문제 풀기

1. 앱을 연다
2. `사진 찍기` 를 눌러 전체화면 촬영을 열거나 사진을 올린다
3. `몇 번 문제 도와줘` 를 말하거나 입력한다
4. OCR이 글자를 읽는다
5. 문제 번호를 고른다
6. 문제 뜻과 생각 포인트를 본다
7. 힌트를 한 단계씩 연다
8. 마지막에만 정답을 연다

### 흐름 B. ChatGPT 보조 사용

1. 앱에서 1차 힌트를 본다
2. 더 깊은 풀이가 필요하면 `ChatGPT 복사`
3. ChatGPT 앱에 붙여넣는다
4. 같은 철학의 추가 코칭을 받는다

### 흐름 C. 최근 기록 다시 열기

1. 아래쪽 `최근 코칭 기록` 영역을 연다
2. 이전에 풀었던 문제를 누른다
3. 같은 힌트와 문제 해석을 다시 본다
4. 필요하면 `기록 지우기` 로 현재 기기 기록만 정리한다

## 2. 화면 구성

- 상단 히어로 영역
- 카메라/사진 영역
- 질문하기 영역
- 문제 해석 및 OCR 결과 영역
- 힌트 영역
- ChatGPT 보조 영역
- 최근 코칭 기록 영역

## 3. 왜 이런 흐름으로 설계했는가

- 아이 입장에서는 복잡한 메뉴보다 한 화면이 편함
- 부모 입장에서는 카메라와 결과가 바로 연결되는 구조가 이해하기 쉬움
- ChatGPT를 완전히 대신하기보다, `로컬 앱 + ChatGPT 보조` 조합이 현실적임

## 4. UX 원칙

- 버튼은 크게
- 한 화면 안에서 끝나게
- 문제 번호 선택은 눈에 띄게
- 상태 메시지는 쉬운 말로
- 소개 키워드는 설명처럼 가볍게
- 실패 시에도 다음 행동을 바로 알 수 있게

## 5. 태블릿 특화 포인트

- 세로 화면 우선
- 손가락 터치가 쉬운 큰 버튼
- 음성 읽기는 힌트 카드와 마지막 단계 카드 안에서 바로 실행
- 홈 화면 추가 가능한 PWA 구조
