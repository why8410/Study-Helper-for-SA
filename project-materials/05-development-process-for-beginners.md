# 개발 진행 설명서

## 1. 이 문서의 목적

이 문서는 개발자가 아닌 사람도
이번 태블릿용 앱을 어떤 순서로 만들었는지 이해할 수 있게 설명하는 문서입니다.

## 2. 실제로 밟은 순서

### 단계 1. 문제 재정의

먼저 확인한 사실:

- 집에 Mac이 없음
- 갤럭시 태블릿에서 써야 함
- 기존 Mac 버전은 그대로 옮길 수 없음

그래서 문제를 다시 정의했습니다.

`Mac용 서비스` 가 아니라
`태블릿에서 열리는 별도 버전` 이 필요하다고 본 것입니다.

### 단계 2. 기술 방향 재선정

Node, Android Studio 같은 무거운 환경 없이도
빨리 움직이는 결과를 만들어야 했습니다.

그래서 선택한 방향:

- 웹앱
- PWA
- 브라우저 OCR
- 브라우저 음성 기능

### 단계 3. 태블릿 화면 설계

아이와 부모가 한 화면 안에서 바로 사용할 수 있게
모든 기능을 한 페이지 안에 넣는 방향으로 설계했습니다.

### 단계 4. 핵심 기능 구현

- 카메라
- 사진 업로드
- 음성 입력
- OCR
- 문제 번호 선택
- 힌트 3단계
- 정답 열기
- 힌트 카드와 마지막 단계 카드 안 음성 읽기
- ChatGPT 복사
- 공유하기
- 최근 기록 저장

### 단계 5. PWA 구조 추가

태블릿 홈 화면에 앱처럼 추가할 수 있게
manifest와 service worker를 붙였습니다.

### 단계 6. 실행과 검증 보조 추가

개발 과정에서 바로 미리볼 수 있게
미리보기 실행 스크립트와 검증 스크립트를 추가했습니다.

이후에는:

- OCR 전처리로 사진 대비를 먼저 보정하고
- 최근 기록을 다시 여는 기능을 넣고
- 공유 버튼으로 ChatGPT 앱 쪽 연결을 더 쉽게 만들었습니다.
- 정적 배포용 릴리스 폴더를 한 번에 만드는 스크립트도 추가했습니다.

### 단계 7. 문서화

나중에 최실장님이 다시 봐도 이해할 수 있게
태블릿 프로젝트 전용 문서 폴더를 만들었습니다.

## 3. 초보자 기준 핵심 이해 포인트

- `PWA`: 웹사이트지만 앱처럼 홈 화면에 추가할 수 있는 형태
- `OCR`: 사진 속 글자를 읽는 기능
- `정적 웹앱`: 별도 서버 프로그램 없이 파일 중심으로 동작하는 앱
- `ChatGPT 복사`: 앱이 직접 GPT를 부르는 대신, 질문을 복사해서 ChatGPT 앱에 붙이는 방식
